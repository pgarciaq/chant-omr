ChantOMR

End-to-end Optical Music Recognition for Gregorian chant

A specialist deep learning model that converts photographs of historical chant manuscripts—antiphonaries, graduals, hymnals—into machine-readable GABC notation. Built on a ConvNeXt-V2 encoder and Transformer decoder, trained on 20,000 GregoBase scores, and deployable via OpenVINO on Intel hardware.

Get Started View on GitHub

What It Does

A ~59M parameter vision-encoder-decoder model purpose-built for Gregorian square notation. Not a general-purpose sheet music recognizer—a specialist that understands neumes, clefs, and four-line staves.

🎶

Photograph to Notation

Feed a photograph of a chant page, get structured GABC notation ready to typeset with Gregorio or import into chant databases. No manual transcription needed.

♦

Square Notation Specialist

Purpose-built for Gregorian neumes on four-line staves—punctum, podatus, clivis, torculus, porrectus, and their variants. Not a repurposed modern-notation or general VLM model.

📚

Synthetic Training Data

Trained on ~20,000 chants from GregoBase, rendered to PNG by Gregorio + LuaLaTeX. CC0 data, fully reproducible pipeline, no manual annotation required.

🎨

Domain Augmentation

On-the-fly aging transforms bridge the gap between clean renders and real manuscripts: parchment texture, foxing, ink fade, perspective skew, and JPEG compression artifacts.

⚡

OpenVINO Deployment

Export the trained model to OpenVINO IR format for fast inference on Intel CPUs, Arc GPUs, and NPUs—no PyTorch runtime needed. Model published on HuggingFace.

🔗

ghh Integration

Plugs into Stage 12 of Guido's Helping Hand—the open-source pipeline that transforms smartphone photos of chant books into searchable PDFs.

Why ChantOMR?

Design decisions that set this project apart from existing approaches.

End-to-End, Not Pipeline

Classical OMR chains staff detection → symbol segmentation → classification → assembly. Each stage propagates errors. ChantOMR learns the full mapping from image to notation in one model.

Specialist, Not VLM

General vision-language models (GPT-4o, Gemini) hallucinate notation and lack GABC knowledge. ChantOMR is a 59M-parameter specialist—1000x smaller, 100x faster, and domain-accurate.

GABC, Not Volpiano

Volpiano strips melismas and omits rhythmic information. GABC preserves full neume structure, liquescence, episema, and text alignment—and is typesetable by Gregorio.

Synthetic + Augmented

No need for expensive manual annotation. Train on 20k GregoBase chants rendered by Gregorio, then bridge the domain gap with on-the-fly augmentation that simulates aged parchment.

Variable-Height Input

Score strips vary in length. ChantOMR accepts variable-height images (fixed width 1050) with a variable-size patch grid—no padding, no cropping, no information loss.

Intel-Native Deployment

Export to OpenVINO IR for inference on Intel Arc GPUs and NPUs. Train on NVIDIA CUDA, deploy on Intel—no vendor lock-in for the inference path.

Metrics & Benchmarks

Target metrics for the v0 release. Evaluation on real manuscript scans will follow once the model is trained and augmentation is production-ready.

Target

GABC Edit Distance

< 30% character-level edit distance between predicted and ground-truth GABC on real manuscript scans. Comparable to Transcoda's 64% OMR-NED on modern notation.

Target

Neume Accuracy

> 80% exact-match accuracy at the neume level: the correct neume glyph on the correct pitch, in the correct position within the syllable.

Target

Structural Validity

> 95% of model outputs are syntactically valid GABC that Gregorio can typeset without errors. Invalid output is worse than slightly wrong output.

Getting Started

Two paths: use the trained model for inference, or train from scratch.

  • Clone and install
    git clone https://github.com/pgarciaq/chant-omr.git
    pip install -e ".[dev]"
  • Download the training corpus
    chant-omr download fetches ~10,000 GABC files from GregoBase (CC0). chant-omr render generates paired PNGs.
  • Train the model
    python scripts/train.py --accelerator cuda on a cloud GPU (RTX 5080 at $0.16/hr, ~24h, ~$4 total). See the Training Guide.
  • Run inference
    chant-omr predict manuscript_page.png decodes a score image into GABC notation.
# Clone the repository
$ git clone https://github.com/pgarciaq/chant-omr.git
$ cd chant-omr && pip install -e ".[dev]"

# Download and render the training corpus
$ chant-omr download
$ chant-omr render

# Train on a CUDA GPU
$ python scripts/train.py --accelerator cuda --precision bf16-mixed

# Run inference on a score image
$ chant-omr predict manuscript_page.png
