ChantOMR

End-to-end Optical Music Recognition for Gregorian chant

A specialist deep learning model that converts photographs of historical chant manuscripts—antiphonaries, graduals, hymnals—into machine-readable GABC notation. Based on Transcoda’s proven ConvNeXt-V2 + Transformer architecture (59M params), retrained from scratch on 20,000 GregoBase scores with domain augmentation for square notation. Deployable via ONNX Runtime or OpenVINO on any hardware.

What It Does

A ~59M parameter vision-encoder-decoder trained from scratch for Gregorian square notation. Not a general-purpose sheet music recognizer—a specialist that understands neumes, clefs, and four-line staves.

🎶

Photograph to Notation

Feed a photograph of a chant page, get structured GABC notation ready to typeset with Gregorio or import into chant databases. No manual transcription needed.

♦

Square Notation Specialist

Purpose-built for Gregorian neumes on four-line staves—punctum, podatus, clivis, torculus, porrectus, and their variants. Not a repurposed modern-notation or general VLM model.

📚

Synthetic Training Data

Trained on ~20,000 chants from GregoBase, rendered to PNG by Gregorio + LuaLaTeX. CC0 data, fully reproducible pipeline, no manual annotation required.

🎨

Domain Augmentation

15 on-the-fly OpenCV transforms bridge the gap between clean renders and real manuscripts: parchment texture overlay (34 real patches), foxing, ink fade, iron gall corrosion, water stains, perspective skew, uneven lighting, and more. See the Augmentation Guide.

⚡

Multi-Hardware Deployment

Export to ONNX for inference on any hardware—NVIDIA, AMD, Intel, Apple Silicon, or CPU—via ONNX Runtime. Optional OpenVINO acceleration for Intel Arc GPUs and NPUs. No PyTorch runtime needed.

🔗

ghh Integration

Why ChantOMR?

Design decisions that set this project apart from existing approaches.

End-to-End, Not Pipeline

Classical OMR chains staff detection → symbol segmentation → classification → assembly. Each stage propagates errors. ChantOMR learns the full mapping from image to notation in one model.

Specialist, Not VLM

General vision-language models (GPT-4o, Gemini) hallucinate notation and lack GABC knowledge. ChantOMR is a 59M-parameter specialist—1000x smaller than a VLM, 100x faster, and domain-accurate.

GABC, Not Volpiano

Volpiano strips melismas and omits rhythmic information. GABC preserves full neume structure, liquescence, episema, and text alignment—and is typesetable by Gregorio.

Synthetic + Augmented

No need for expensive manual annotation. Train on 20k GregoBase chants rendered by Gregorio, then bridge the domain gap with on-the-fly augmentation that simulates aged parchment.

Variable-Height Input

Score strips vary in length. ChantOMR accepts variable-height images (fixed width 1050) with a variable-size patch grid—no padding, no cropping, no information loss.

Hardware-Agnostic Deployment

Train on NVIDIA CUDA, deploy everywhere. Export to ONNX for inference on any hardware via ONNX Runtime, with optional OpenVINO acceleration for Intel Arc GPUs and NPUs.

Metrics & Benchmarks

Evaluated on the synthetic test split (1,000 clean rendered scores, beam_width=3). Trained with domain augmentation (15 transforms). These are held-out samples the model never trained on.

3.1%

GABC Edit Distance

3.1% mean normalized edit distance (target: < 30%). The vast majority of samples score below 5%. Encoding-equivalence normalization ensures fair comparison.

97.0%

Neume Accuracy

97.0% equiv-normalized neume accuracy (target: > 80%). The model reliably recognizes pitch, glyph shape, liquescence, and syllable–neume alignment.

97.7%

Structural Validity

97.7% of outputs are syntactically valid GABC (target: > 95%). Grammar-constrained decoding and Gregorio compilation checks are available to further improve edge cases.

99.0%

Gregorio Compilation

99.0% of predictions compile successfully through the Gregorio binary—the gold-standard structural validator. Only 10 failures in 1,000 samples.

Note: evaluated on clean synthetic renders. Real manuscript performance will be validated separately once annotated manuscript test data is available.

Getting Started

Use the pre-trained model from HuggingFace, or train from scratch on your own data.

  • Install chant-omr
    pip install chant-omr
  • Run inference with the HuggingFace model
    chant-omr predict score.png --model pgquiles/chant-omr downloads the model automatically and outputs GABC notation.
  • Choose your backend
    Add --device openvino for Intel Arc GPUs, --device onnx for any hardware, or leave as auto for PyTorch (CUDA/XPU/CPU).
  • Or train from scratch
    chant-omr download + chant-omr render + python scripts/train.py --accelerator cuda. See the Training Guide.
# Install and run with the pre-trained model
$ pip install chant-omr
$ chant-omr predict score.png --model pgquiles/chant-omr

# Or with a specific backend
$ chant-omr predict score.png --model pgquiles/chant-omr --device openvino

# Or train from scratch
$ git clone https://github.com/pgarciaq/chant-omr.git
$ cd chant-omr && pip install -e ".[dev]"
$ chant-omr download && chant-omr render
$ python scripts/train.py --accelerator cuda --precision bf16-mixed
